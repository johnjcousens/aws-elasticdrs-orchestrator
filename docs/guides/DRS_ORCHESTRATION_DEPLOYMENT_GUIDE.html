Overview

This guide discusses architectural considerations and configuration steps for deploying AWS DRS Orchestration, an enterprise-grade disaster recovery orchestration platform for AWS Elastic Disaster Recovery (DRS). It provides links for launching AWS CloudFormation templates that automate the deployment.

The guide is for IT infrastructure architects and disaster recovery specialists who want to design and deploy a solution for orchestrating complex multi-tier application recovery with wave-based execution, dependency management, and automated health checks.

AWS DRS Orchestration enables organizations to orchestrate complex multi-tier application recovery with wave-based execution, dependency management, and automated health checks using AWS-native serverless services. The solution provides:

  • Wave-Based Recovery: Execute disaster recovery in coordinated waves with explicit dependencies between tiers

  • Protection Groups: Organize DRS source servers into logical groups for coordinated recovery

  • Drill Mode: Test recovery procedures without impacting production environments

  • Real-Time Monitoring: Track execution progress with detailed status updates

  • API-First Design: Complete REST API for DevOps integration and automation workflows

Key Capabilities

  • Cost-Effective: $12-40/month operational cost with pay-per-use serverless pricing

  • Unlimited Waves: Flexible wave-based orchestration with no artificial constraints

  • Platform Agnostic: Supports any source platform (VMware, physical servers, cloud)

  • Sub-Second RPO: Leverages AWS DRS continuous replication capabilities

  • Fully Serverless: No infrastructure to manage, scales automatically

Deployment options

This solution provides automated deployment through AWS CloudFormation templates. Choose the deployment option that best fits your environment:

  • New VPC Deployment: Deploy the complete solution including a new VPC with public and private subnets

  • Existing VPC Deployment: Deploy into your existing VPC infrastructure

AWS DRS is available in 30 AWS regions (28 commercial + 2 GovCloud). Ensure DRS is available in your target region before deployment.

Plan the deployment

Before deploying the solution, consider the following planning aspects:

DRS Prerequisites

  • DRS Initialization: AWS DRS must be initialized in your target region

  • Source Servers: Source servers should be replicating to AWS DRS

  • IAM Permissions: Appropriate IAM permissions for DRS operations

  • Network Connectivity: Ensure source servers can communicate with AWS DRS

Predeployment steps

Complete these steps before deploying the solution:

  1. AWS Account Setup: Ensure you have an AWS account with appropriate permissions

  2. Region Selection: Choose an AWS region where DRS is available

  3. DRS Initialization: Initialize AWS DRS in your target region

  4. S3 Bucket: Create an S3 bucket for deployment artifacts

  5. Service Quotas: Request quota increases if needed for your deployment scale

Ensure AWS DRS is initialized and source servers are replicating before deploying the orchestration solution.

  1. Sign in to your AWS account and navigate to the CloudFormation console

  2. Choose the correct AWS Region where DRS is initialized

  3. Create a new stack using the master template URL

  4. Configure stack parameters:

    • ProjectName: Name for your DRS orchestration deployment

    • Environment: Environment identifier (dev, test, prod)

    • SourceBucket: S3 bucket containing deployment artifacts

    • AdminEmail: Email for the initial admin user

  5. Review and acknowledge IAM resource creation

  6. Deploy the stack (takes approximately 20-30 minutes)

  7. Monitor deployment progress in the CloudFormation console

  8. Retrieve stack outputs for application URLs and configuration

Example Deployment Command

aws cloudformation deploy \
  --template-url https://your-bucket.s3.us-east-1.amazonaws.com/cfn/master-template.yaml \
  --stack-name drs-orchestration \
  --parameter-overrides \
    ProjectName=drs-orchestration \
    Environment=prod \
    SourceBucket=your-bucket \
    AdminEmail=admin@yourcompany.com \
  --capabilities CAPABILITY_IAM CAPABILITY_NAMED_IAM \
  --region us-east-1

Postdeployment steps

After successful deployment, complete these configuration steps:

Create Admin User

USER_POOL_ID=$(aws cloudformation describe-stacks \
  --stack-name drs-orchestration \
  --query 'Stacks[0].Outputs[?OutputKey==`UserPoolId`].OutputValue' \
  --output text)

aws cognito-idp admin-create-user \
  --user-pool-id $USER_POOL_ID \
  --username admin@yourcompany.com \
  --user-attributes Name=email,Value=admin@yourcompany.com Name=email_verified,Value=true \
  --temporary-password "TempPass123!" \
  --message-action SUPPRESS

Access the Application

  1. Retrieve the CloudFront URL from stack outputs

  2. Navigate to the application URL

  3. Sign in with the admin credentials

  4. Complete the initial setup wizard

Configure Protection Groups

  1. Navigate to Protection Groups in the application

  2. Create protection groups for your server tiers

  3. Assign DRS source servers to appropriate groups

  4. Verify server assignments and status

Security

The solution implements comprehensive security controls:

  • Encryption at Rest: All data encrypted in DynamoDB and S3

  • Encryption in Transit: HTTPS enforced via CloudFront and API Gateway

  • Authentication: Cognito JWT token-based authentication

  • Authorization: IAM least-privilege policies

  • Network Security: Security groups with minimal required access

  • Audit Logging: CloudTrail integration for API calls

IAM Permissions

The solution requires specific IAM permissions for DRS operations. Key permissions include:

  • drs:DescribeSourceServers

  • drs:StartRecovery

  • drs:DescribeJobs

  • drs:TerminateRecoveryInstances

  • ec2:DescribeInstances

  • ec2:StartInstances
